# Battle in the Whispering forest

Есть две фракции - южане(армия Джейме Ланнистера) и северяне(армия Робба старка).

Все юниты имеют общий интерфейс, но могут отличаться реализацией, например, по-разному атакуют других юнитов, имеют разное начальное состояние здоровья. Чтобы не привязывать код программы к конкретным классам, не применять каждый раз условные операторы, а также иметь возможность легко добавлять новые типы юнитов, в Unit.h реализован паттерн фабричный метод: классы Cavalryman, Infantryman и Agent унаследованы от класса Unit.

Ещё одна проблема: мы хотим иметь две фракции и чтобы каждая фракция имела своих юнитов, которые могут различаться, например, южан-пехотинцев, серерян-пехотинцев, южан-наездников, северян-наездников и так далее. Создавать семейства связанных объектов и не привязываться к конкретным классам создаваемых нами объектов позволяет паттерн абстрактная фабрика, который в будущем сможет упростить добавление новых наборов юнитов в программу, если мы захотим расширяться. Итак, в UnitFactories.h создаётся абстрактная фабрика FractionFactory, от которой наследуются конкретные фабрики Northerners и Southerners. Они создают юнитов при помощи методов CreateCavalryman(), CreateInfantryman(), CreateAgent(). Cavalryman, Infantryman, Agent от двух фракций - конкретные продукты, связанные по смыслу и являющиеся наследниками классов Cavalryman, Infantryman и Agent, которые можно считать абстрактными продуктами.

Параметры запуска программы:

mkdir build  
cd build  
cmake <path/to/dir>/game  
make  
./game

Тестирование. В первой части два набора тестов - на начальное состояние отдельных юнитов и на фабрики.

Параметры запуска тестов:

mkdir build  
cd build  
cmake <path/to/dir>/tests  
make  
./tests
