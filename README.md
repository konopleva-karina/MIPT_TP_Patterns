# Battle in the Whispering forest

Есть две фракции - южане(армия Джейме Ланнистера) и северяне(армия Робба старка).

Все юниты имеют общий интерфейс, но могут отличаться реализацией, например, по-разному атакуют других юнитов, имеют разное начальное состояние здоровья. Чтобы не привязывать код программы к конкретным классам, не применять каждый раз условные операторы, а также иметь возможность легко добавлять новые типы юнитов, в Unit.h реализован паттерн фабричный метод: классы Cavalryman, Infantryman и Agent унаследованы от класса Unit.

Ещё одна проблема: мы хотим иметь две фракции и чтобы каждая фракция имела свои наборы юнитов, которые могут различаться, например, армию южан-пехотинцев, серерян-пехотинцев, южан-наездников, северян-наездников и так далее. Создавать семейства связанных объектов и не привязываться к конкретным классам создаваемых нами объектов позволяет паттерн абстрактная фабрика, который в будущем сможет упростить добавление новых наборов юнитов в программу, если мы захотим расширяться. Итак, в UnitFactories.h создаётся абстрактная фабрика FractionFactory, от которой наследуются конкретные фабрики Northerners и Southerners. Они создают наборы юнитов при помощи методов CreateCavalrymen(), CreateInfantrymen(), CreateAgents(). Cavalrymen, Infantrymen, Agents от двух фракций - конкрутные продукты, связанные по смыслу и включающие в себя классы Cavalryman, Infantryman и Agent, которые можно считать абстрактными продуктами.

Параметры запуска программы:

> > mkdir build
> > cd build
> > cmake <path/to/dir>/game
> > make
> > ./game

Тестирование. В первой части два набора тестов - на отдельных юнитов(их начальное состояние) и на фабрики(начальная комплектация армий).

Параметры запуска тестов:

> > mkdir build
> > cd build
> > cmake <path/to/dir>/tests
> > tests
> > ./tests
